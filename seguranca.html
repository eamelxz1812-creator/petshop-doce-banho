Segurança e Proteção de Dados

Resumo prático para proteger o site e os dados dos clientes:

  • HTTPS obrigatório: publique em serviços que forneçam SSL (GitHub Pages, Netlify, Vercel).
  • Não exponha segredos: mantenha chaves e credenciais fora do repositório (.env e .gitignore).
  • Valide e sanitize dados: evite SQL injection e XSS usando consultas parametrizadas e escape de saída.
  • Autenticação forte: use provedores como Supabase/Firebase para login com senha segura e 2FA para acessos administrativos.
  • Backups e logs: configure backups periódicos e monitore acessos suspeitos.
  • Permissões mínimas: se houver painel, crie contas com permissões separadas para funcionários.
  • LGPD: colete apenas o necessário e documente como os dados são usados (Política de Privacidade).

Se quiser, posso gerar um checklist técnico para implementar cada item com exemplos (Node, PHP, Supabase, etc.).
